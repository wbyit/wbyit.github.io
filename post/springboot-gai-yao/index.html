Spring Boot 介绍

多年年以来， Spring 平台饱受⾮非议的⼀一点就是⼤大量量的 XML 配置以及复杂的依赖管理理

随着使⽤用 Spring 进⾏行行开发的个⼈人和企业越来越多， Spring 也慢慢从⼀一个单⼀一简洁的⼩小框架变成⼀一个⼤大⽽而全的开源软件， Spring 的边界不不断的进⾏行行扩充，到了了后来 Spring 几乎可以做任何事情了，市⾯面上主流的开源软件、中间件都有 Spring 对应组件⽀支持，人们在享用 Spring 的便利之后，也遇到了一些问题

Spring 每集成⼀个开源软件，就需要增加一些基础配置，慢慢的随着⼈们开发的项⽬越来越庞大，往需要
集成很多开源软件，后期使用 Spring 开发⼤大型项目需要引入很多配置文件，太多的配置非常难以理解，并容易配置出错，到了了后来⼈们甚⾄至称 Spring 为配置地狱

2013 年年，微服务的概念也慢慢兴起，快速开发微⼩独⽴的应用变得更为急迫， Spring 刚好处在这么⼀一个交叉点上，于 2013 年年初开始的 Spring Boot 项目的研发， 2014 年， Spring Boot 伴随着 Spring 4.0 诞⽣生发布了第⼀个正式版本

Spring Boot 并不是要成为 Spring 平台⾥⾯众多“Foundation”层项目的替代者。 Spring Boot的⽬标不在于为已解决的问题域提供新的解决方案，而是为平台带来另一种开发体验，从而简化对这些已有技术的使用。对于已经熟悉 Spring 生态系统的开发人员来说， Spring Boot 是一个很理想的选择；对于采用 Spring 技术的新人来说， Spring Boot 提供一种更简洁的方式来使用这些技术

  • 随着动态语言的流行，java的开发显得格外的笨重；繁多的配置、低下的开发效率，负责的部署流程以及第三方技术整合难度大。
  • 在上述环境下，Spring Boot由此诞生，它的设计是为了使您能够尽可能快地启动和运行。它使用“习惯优于配置”的理念让你的项目快速运行起来。使用Spring Boot很容易创建一个独立运行、准生产强力的基于Spring框架的项目，使用Spring Boot你可以不用或者只需要很少的Spring配置。
  • 提供了J2EE开发的一站式解决方案。

Spring Boot 的优点

  • 快速构建独立运行的Spring项目；
  • 无须依赖外部Servlet容器，应用无需打成WAR包；项目可以打成jar包独自运行；
  • 提供 一系列 starter pom 来简化 Maven 的依赖加载；
  • 大量的自动配置，对主流开发框架的无配置集成；
  • 无须配置XML，开箱即用，简化开发，同时也可以修改默认值来满足特定的需求；
  • Spring Boot 并不是对 Spring 功能上的增强，而是提供了一种快速使用 Spring 的方式；
  • 极大提高了开发、部署效率。
